Рекомендованная архитектура данных для PWA

Учитывая ваши требования (Offline Mode, Fuzzy Search, Favorites), я не рекомендую делать прямые запросы к внешним API в реальном времени. Вместо этого:

    ETL-процесс (Extract, Transform, Load): Напишите скрипт (например, на Node.js), который:

        Выкачивает данные из Wikidata (SPARQL) и Trefle.

        Переводит недостающие поля на русский (если нужно).

        Формирует единый JSON-файл (plants_db.json).

    Внедрение в PWA:

        Поместите этот JSON в assets вашего приложения.

        При первом запуске загружайте данные в IndexedDB (так как localStorage имеет лимит 5-10 МБ, что мало для базы с рецептами и описаниями).

        Поиск (Fuzzy search) будет работать очень быстро прямо в браузере по IndexedDB.

Техническое предостережение (Disclaimer)

Так как приложение касается здоровья:

    Убедитесь, что в данных, которые вы соберете, есть четкие пометки о токсичности. В Trefle и Wikidata есть флаги toxicity. Обязательно добавьте дисклеймер в интерфейс приложения о том, что информация носит справочный характер.

Для вашего проекта правильный «перевод» — это не совсем перевод (translation), а скорее сопоставление понятий (mapping). Если вы просто прогоните названия растений через Google Translate, вы получите ошибки: Lime станет «известью» (вместо «лайма» или «липы»), а Sage — «мудрецом» (вместо «шалфея»).

Для open source проекта лучшая стратегия — комбинированная.
Стратегия перевода: «3 слоя»
1. Слой сущностей (Названия растений и болезней)

Не переводите их. Получайте уже существующие русские названия из Wikidata. Это самый надежный способ.

    Инструмент: SPARQL-запросы к Wikidata.

    Почему: У каждого растения есть уникальный ID (например, Ромашка аптечная — Q28437). У этого ID есть метки (label) на разных языках. Вы просто запрашиваете метку ru.

Пример SPARQL-запроса для вашего PWA: Этот запрос можно выполнить на query.wikidata.org и скачать результат в JSON.
Code snippet

SELECT ?plant ?plantLabel ?taxonName ?medicalConditionLabel WHERE {
  ?plant wdt:P31 wd:Q188.      # Ищем сущности, являющиеся "лекарственным растением"
  ?plant wdt:P225 ?taxonName.  # Получаем научное название (латынь)
  
  OPTIONAL { 
    ?plant wdt:P2175 ?medicalCondition. # Если есть информация, что оно лечит
  }

  # Магия перевода: сервис автоматически дает русские названия, 
  # а если их нет — английские, или оставляет ID
  SERVICE wikibase:label { bd:serviceParam wikibase:language "ru,en". }
}
LIMIT 1000

2. Слой интерфейса (Кнопки, меню, категории)

Здесь объем текста мал, поэтому используйте ручной перевод или готовые глоссарии.

    Для категорий болезней (Respiratory, Digestive и т.д.) используйте официальные медицинские термины (МКБ-10 — Международная классификация болезней), чтобы звучать профессионально.

        Respiratory -> Заболевания органов дыхания

        Digestive -> Заболевания органов пищеварения

3. Слой контента (Рецепты, описания, советы)

Это самый объемный пласт. Здесь придется использовать машинный перевод, но с последующей валидацией.

    Лучший бесплатный инструмент (Offline): Библиотека Argos Translate (Python) или модель Helsinki-NLP/opus-mt-en-ru (через Hugging Face). Они работают офлайн и бесплатны.

    Лучший платный (API): DeepL API (у них есть бесплатный уровень 500,000 символов в месяц — этого хватит для старта). Он переводит медицинские тексты намного точнее Google.

Практический Workflow (Как это сделать)

    Скачайте базу на английском (как обсуждали выше, например, из Trefle или Kaggle).

    Скрипт сопоставления (Node.js/Python):

        Берет латинское название из вашей базы (например, Matricaria chamomilla).

        Ищет его в Wikidata через API.

        Забирает поле ru_label ("Ромашка аптечная").

    Машинный перевод описаний:

        Оставшиеся текстовые поля (Description, Recipe) прогоняете через скрипт с opus-mt-en-ru.

    Валидация (Checklist):

        Проверьте "ложных друзей переводчика" (Angina в английском часто "стенокардия", а не только "ангина/тонзиллит").

        Обязательно добавьте дисклеймер: "Перевод выполнен автоматически. Проконсультируйтесь с врачом".
